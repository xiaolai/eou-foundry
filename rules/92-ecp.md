# EOU Change Proposal Rule

All significant Foundry mutations must be represented as an ECP.

Use an ECP when changing:

- EOU purpose
- authority level
- stop conditions
- validators
- schema fields
- promotion or retirement rules
- responsibility ownership
- generation envelope
- constitution or governance rules

An ECP must include observed problem, failure class, proposed change, risks, tests required, simulation status, approval status, and rollback path when applicable.
